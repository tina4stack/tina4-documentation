Basic Routing

Tina4 discovers routes automatically. Drop a Python file in src/routes/ and the framework picks it up — no registration, no imports, no config.

Define a route

from tina4_python import get

@get("/hello")
async def hello(request, response):
    return response("Hello, world!")

Every route handler receives two arguments: the request and a response callable. Return via response() — that is the Tina4 convention.

Route parameters

@get("/users/{id}")
async def get_user(request, response):
    user_id = request.params["id"]
    return response(f"User {user_id}")

Curly-brace placeholders capture URL segments. They appear in request.params as a dictionary.

HTTP methods

Import the decorator for the method you need: get, post, put, patch, delete.
